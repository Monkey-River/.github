# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in any Monkey & River project, please report it responsibly. **Do not open a public GitHub issue.**

Send your report to: **info@monkeyandriver.com**

Include as much of the following as possible:

- A description of the vulnerability and its potential impact
- The affected repository and version (if known)
- Steps to reproduce or a proof-of-concept
- Any suggested mitigations

## What to Expect

We aim to acknowledge reports within **2 business days** and will keep you informed as we investigate and address the issue. We ask that you give us a reasonable window to resolve the vulnerability before any public disclosure.

## Scope

This policy applies to all repositories under the [Monkey-River](https://github.com/Monkey-River) GitHub organisation.

## Out of Scope

- Vulnerabilities in third-party dependencies should be reported directly to the upstream maintainer
- Issues in client repositories that are not managed by Monkey & River

## Thank You

We appreciate responsible disclosure and the effort it takes to report security issues. If you have questions about this policy, contact us at info@monkeyandriver.com.
